```go
// 熔断配置示例
sentinel:
  rules:
    flow:
      - resource: "POST:/api/v1/memories"
        threshold: 100        # QPS 阈值
        grade: 1              # QPS 模式
        limitApp: default
        
      - resource: "GET:/api/v1/nfts/{id}"
        threshold: 500
        grade: 1
        limitApp: default
    
    circuit-breaker:
      - resource: "ai-service-call"
        strategy: 2           # 错误率策略
        retryTimeoutMs: 5000  # 熔断时长
        minRequestAmount: 10  # 最小请求数
        statIntervalMs: 10000 # 统计时长
        slowRatioThreshold: 0.3  # 慢调用比例
```

```
│  • 客户端存储 (Secure Storage)   • 可撤销 (logout 时)         │
│  • 无状态验证                    • 有状态管理                │
│                                                              │
│  刷新流程：                                                 │
│  1. Access Token 过期 → 401 Unauthorized                    │
│  2. 客户端使用 Refresh Token 请求刷新                        │
│  3. 服务端验证 Redis 中的 Refresh Token                      │
│  4. 签发新的 Access Token + Refresh Token                   │
└─────────────────────────────────────────────────────────────┘
```

```
数据库:
• 读写分离 (读流量走 Replica)
```
